# 库存补货 SOP（示例/合成数据）

## 安全库存
- 每个 SKU 设有**安全库存线 (safety stock)**，默认 **30 件**。
- 当前库存 **低于安全库存**时，应触发补货流程并生成运营工单。

## 补货流程
1. 系统监测库存低于安全线 → 标记为待补货。
2. 运营创建补货工单，注明 SKU、当前库存、建议补货量。
3. 联系供应商确认交期（见 vendor_management_policy）。

## 缺货风险
- 库存 < 安全线且为热销/广告投放中的 SKU，属于**高优先级**，需当日处理。
- 长期缺货商品应考虑暂停相关广告投放，避免预算浪费。
