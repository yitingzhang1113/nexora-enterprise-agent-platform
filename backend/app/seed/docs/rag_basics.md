# 检索增强生成 (RAG) 基础知识（示例资料）

## 什么是 RAG
RAG（Retrieval-Augmented Generation）在生成回答前，先从知识库检索相关资料，
把资料作为上下文喂给大模型，从而让回答有依据、可溯源，减少幻觉。

## 典型流程
1. 文档入库：解析 → 清洗 → 切块 → 嵌入 → 写入向量库。
2. 检索：把用户问题向量化，做向量近邻检索；常配合关键词检索做混合召回。
3. 重排：对召回结果按相关性重新排序，保留最相关的若干片段。
4. 生成：把片段拼成上下文，连同问题一起交给大模型流式生成答案。

## 混合检索与 RRF
向量检索擅长语义相似，关键词检索（如 BM25）擅长精确术语。把两路结果用
RRF（Reciprocal Rank Fusion，按排名倒数求和）融合，能兼顾语义与精确匹配。

## 向量库
常见向量库有 Milvus、Qdrant、Weaviate、pgvector 等，用于存储与近邻检索嵌入向量。
